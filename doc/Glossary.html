Glossary

Intrinsic Accessibility
By implementing customizable application objects with plugins (insofar as they can be customized), the developer maximizes accessibility while minimizing development effort. See this principle preached and practiced.

Minimal Drift
In any development environment, the user will be required at times to drift away from the task at hand in pursuit of some information that is necessary for their progress. I take the effectiveness of a development environment to be a direct measure of the frequency of these interruptions, the number of steps required to be taken away from the editor, and the potential for getting lost at each step. See the minimization of drift practiced and preached.

Java API
The Java API contains almost the entire set of facilities provided by a Java Runtime Environment. Only its primitive types are treated as langauge constructs, and even those have handles in the API.

Java Runtime Environment
The JRE contains the Java Virtual Machine and a copy of the class library that comprises the Java API. Its download is about 10MB, and it is available for commercial redistribution.

Java Virtual Machine
The JVM is the binary executable in which the Java language runs. It may be loaded in the address space of any application, which may then call methods on Java object instances in that companion JVM. A member of the family Suidae, the JVM is weighty and uncompromising in its use of system resources.

Java Native Interface
Java was not originally designed to run in conjunction with other platform-specific code. When the need arose, the keyword native was added to the language to refer to methods implemented in another language. Such implementations can only be accessed by Java when exported from a dynamic library. The JRE comes with a static library jvm.lib that provides C methods to instantiate and run the JVM, and along with various (usually cumbersome) means of interacting with it. The JNI is discussed (somewhat vaguely) in the Java Tutorial. While the JNI is not the most elegant aspect of Java, it does in fact provide sufficient means for integrating Java with native code -- it's just a hassle to work with it. See the source code for the static library plugin.o for an example of what is required.

Garbage Collector
When the reference count on a Java object goes to zero, the JVM marks it as disposable, such that a system process known as the garbage collector will eventually locate the object and free its memory. One of the hassles of Java is that the garbage collector is neither efficient nor accurate; if memory is at a premium and plugins are run repeatedly, Decaf will have to include some mechanism for dumping the JVM and reloading a new instance of it.

KJC
An open-source Java compiler from the Kopi project, which is covered under the GNU General Public License.

Jacks
This hip and masculine term refers to a compiler regression test from IBM that verifies the degree to which a compiler conforms to the Java Language Specification. If a compiler has "passed" Jacks, it definitively maintains each and every requirement of the JLS.

Bytecode
The term bytecode refers to a stream of bytes that is understood by the physical processor of a computer. Java source files are compiled into Java bytecode, which is interpreted by the Java Virtual Machine when the code is executed or referenced. Java bytecode differs from pure (processor-specific) bytecode in that it has been abstracted to run on a number of processors: Intel (PC), Alpha (PC), Motorola (Mac), SPARC (Unix), AIX (Unix), etc., and it requires the JVM to translate it into pure bytecode as it runs.

Third Generation Language
The term 3GL refers to programming languages that are compiled, maintain strict rules about data types, and usually provide substantial structure. They are known as the third generation because they compile into assembly language constructs (the second generation), which are further compiled into some variety of bytecode (the first generation). Common examples are Java, C and C++.

Fourth Generation Language
The term 4GL refers to programming languages that either compile into a 3GL, or that are tranlated into bytecode as they run (the latter are known as scripting languages).

Token
The term token is used by compilers to refer to a set of characters delimited by whitespace (i.e., space, tab, and newline) and/or structural characters (i.e., semicolon, parenthese, quotation mark, comma, etc.). Simply put, a token is a "word" of source code. In Java, a token represents either a language keyword, a class name or member of a class (i.e., a field, method, or inner class).

Parse
The first step in compiling a file of source code is to parse the file, identifying each token as a language keyword, a reference to a data structure, or unknown (i.e., a parse error).

Alpha
An alpha model of a software product in development exhibits the fundamental object model of the finished product, but lacks implementation of features within that framework, and is typically full of glitches, errors, hassles, undefined conditions, hang points, unexpected behaviors, display discrepancies, inconsistent responses, incomprehensible user messages ... that is to say, bugs.
